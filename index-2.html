Modules and Architecture

Project organized into several modules, each responsible for different aspects of the simulation. This modular architecture makes the code reusable flexible and extensible, allowing for easier maintenance and development.

    • envelope

      Takes a Segment and creates a Polygon around it. Used to convert skeleton segments into 2D polygons.

    • point

      Takes x and y coordinates to represent a point in 2D space, with methods for calculating distances and transformations.

    • polygon

      Takes an array of Point objects representing vertices. Provides methods for calculating area, perimeter, and determining point inclusion.

    • segment

      Takes two Point objects to define the endpoints of the segment. Includes methods for detecting intersections and calculating length.

    • cross

      Defines crosswalk markings for pedestrian interactions in the simulation.

    • light

      Handles traffic light markings and their placement in the simulation environment.

    • parking

      Defines parking space markings for car navigation and parking behaviors.

    • start

      Represents the start line marking for the car's journey.

    • stop

      Defines stop line markings for intersections and stopping points.

    • target

      Defines target points for navigation and movement objectives.

    • yield

      Handles yield markings for traffic merging scenarios.

    • beepAudio

      Manages audio feedback, specifically the beep sounds used during various simulation events.

    • engineAudio

      Simulates engine sounds to make the driving experience more realistic.

    • dispatcher

      Acts as the main event dispatcher, handling communication between different components of the simulation.

    • marking

      Manages road markings, which are used for guiding the car within lanes and ensuring adherence to traffic rules.

    • markingEditor

      Provides editing tools for road markings, including placement and adjustments.

    • brainControls

      Contains the main logic for decision-making, representing the 'brain' of the car.

    • cameraControl

      Enables user or AI control over the camera, including zoom and angle adjustments.

    • dummyControls

      Handles dummy car controls for simple simulations.

    • keyboardControls

      Enables controlling the car using the keyboard for manual simulations.

    • phoneControls

      Handles car controls using a phone, allowing mobile input for the simulation.

    • crossEditor

      Provides tools to edit crosswalks within the simulation, used to simulate pedestrian interactions.

    • graphEditor

      Provides tools to edit graph-based navigation paths for the car.

    • lightEditor

      Handles editing of traffic lights within the simulation environment.

    • parkingEditor

      Provides tools to edit parking spaces and related behaviors in the simulation.

    • startEditor

      Enables editing of starting positions for the car in the simulation.

    • stopEditor

      Handles editing of stop points for the car within the simulation.

    • targetEditor

      Allows editing of target locations for the car to navigate towards.

    • yieldEditor

      Handles editing of yield signs and related car behaviors.

    • car

      Defines the car object, its properties, and behaviors, including acceleration, steering, and collision detection.

    • sensor

      Simulates the sensors mounted on the car, allowing it to detect obstacles and navigate the environment.

    • building

      Defines buildings in the environment, including their properties and spatial relationships.

    • network

      Handles the network structure for road connections and car navigation.

    • primitive-road

      Defines simple road structures for use in the simulation environment.

    • tree

      Represents trees in the environment, adding realism to the simulation.

    • graph

      Handles graph-based algorithms for pathfinding and other spatial calculations.

    • markerDetector

      Detects and processes markers for use in navigation and positioning.

    • osm

      Handles integration with OpenStreetMap (OSM) data for realistic navigation scenarios.

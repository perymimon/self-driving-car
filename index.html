Self-Driving Car Project

The main objective of this project is to develop a fundamental self-driving car simulation from scratch. We utilize HTML, CSS, and JavaScript to create an environment in which a virtual car navigates autonomously, using simulated sensors and simple decision-making logic. This project aims to introduce learners to key concepts such as sensor fusion, pathfinding, and basic machine learning techniques for autonomous vehicles.

Modules and Architecture

The Self-Driving Car Project is organized into several modules, each responsible for different aspects of the simulation. This modular architecture makes the project flexible and extensible, allowing for easier maintenance and development.

Controls

  • dummyControls.js - Placeholder for basic, dummy controls used for testing.
  • keyboardControls.js - Handles car control via keyboard input.
  • phoneControls.js - Manages controls using a phone's sensors.

Editors

  • crossEditor.js - Editor for managing cross sections in the world.
  • graphEditor.js - Graph editor for defining paths and connections.
  • lightEditor.js - Editor for managing traffic lights within the environment.
  • parkingEditor.js - Editor for setting up parking spots.
  • startEditor.js - Defines starting points for the car in the simulation.
  • stopEditor.js - Defines stopping points or end conditions for the simulation.
  • targetEditor.js - Manages target locations or goals within the world.
  • yieldEditor.js - Editor for setting up yield signs and their behaviors.

Items

  • building.js - Represents buildings in the simulation environment.
  • car.js - Core module defining the car's behavior and properties.
  • network.js - Handles the network connections between various nodes in the environment.
  • sensor.js - Defines and manages the sensors attached to the car.
  • tree.js - Represents trees in the simulation environment.

Markings

  • cross.js - Manages crosswalks and similar markings on roads.
  • light.js - Handles the visual and functional aspects of traffic lights.
  • parking.js - Manages parking spot markings.
  • start.js - Defines starting line or position markings.
  • stop.js - Defines stop line or position markings.
  • target.js - Manages target or goal markings.
  • yield.js - Handles yield sign markings.

Math

  • graph.js - Core graph algorithms and utilities for pathfinding.
  • markerDetector.js - Simple math-based marker detection module.
  • osm.js - Handles OpenStreetMap data integration.
  • region.js - Manages regions within the world, likely for pathfinding or area-based logic.

Primitives

  • envelope.js - Mathematical primitive for envelope shapes.
  • point.js - Basic point primitive, likely used in geometry calculations.
  • polygon.js - Handles polygon shapes and their properties.
  • segment.js - Defines segments, crucial for line intersection calculations.

Utils

  • 3d-utils.js - Utilities for managing 3D aspects of the simulation.
  • algebra-math-utils.js - General algebra and math utilities.
  • canvas-utils.js - Utilities for canvas drawing and manipulation.
  • codeflow-utils.js - Handles code flow logic, likely for asynchronous tasks.
  • math-utils.js - General math utilities, used across different modules.

BrainVisualizer

  • miniMap.js - Handles the creation and display of a minimap.
